Java基础之面向对象

Author: ChenJunYu Date: February 7, 2021  17:26:00 Category: Java基础

什么是对象

就是将现实世界中的实体抽象成对象，每一个实体就是一个对象，有着自己独特的特性。比如：车就是一个对象。

什么是面向对象

面向对象(Object Oriented)是一种对现实世界理解和抽象的方法，是计算机编程技术发展到一定阶段后的产物。面向对象是相对于面向过程来讲的，面向对象方法，把相关的数据和方法组织为一个整体来看待，从更高的层次来进行系统建模，更贴近事物的自然运行模式。

就是将问题分解成一个个步骤，对每个步骤抽象成相应的对象，再通过这不同对象之间的调用来解决问题。也就是说，一个对象会有他的属性和行为，通过这些对象来进行业务逻辑的实现。例如：定义一个车Car类，封装了这个车Car类的属性(颜色、型号等)和行为(前进、后退等)

什么是面向过程

面向过程(Procedure Oriented)是一种以过程为中心的编程思想。这些都是以什么正在发生为主要目标进行编程，不同于面向对象的是谁在受影响。与面向对象明显的不同就是封装、继承、类。简写为POP。

就是将问题分解成一个个步骤，然后使用函数将这些步骤一一实现，使用的时候一个一个依次调用。最典型的就是实现一个算法，例如实现一个排序方式。

面向对象和面向过程的区别

举个最简单的例子：完成一个五子棋

面向过程：1、开始游戏，2、黑子先走，3、绘制画面，4、判断输赢，5、轮到白子，6、绘制画面，7、判断输赢，8、返回步骤2，9、输出最后结果。把上面每个步骤用不同的方法来实现。

面向对象：1、构建黑白棋子，两个行为是一模一样的。2、构建棋盘对象，负责绘制棋盘。3、构建规则，负责定义输赢等。第一类对象（玩家对象）负责接受用户输入，并告知第二类对象（棋盘对象）棋子布局的变化，棋盘对象接收到了棋子的变化就要负责在屏幕上面显示出这种变化，同时利用第三类对象（规则系统）来对棋局进行判定。

优缺点

面向过程

优点：性能比面向对象高，因为类调用时需要实例化，开销比较大，比较消耗资源。
缺点：没有面向对象易维护、易复用、易扩展

面向对象

优点：易维护、易复用、易扩展，由于面向对象有封装、继承、多态性的特性，可以设计出低耦合的系统，使系统 更加灵活、更加易于维护
缺点：性能比面向过程低

面向对象的三大基本特征

封装(Encapsulation)

封装就是指将某一事物封装成一个抽象的类，类中包含了这个事物的特点与行为，类的定义可以看成是将构成类的成员变量和方法封装起来。将类的某些信息隐藏在类的内部，不允许外部程序直接访问，并通过该类提供的方法来实现对隐藏信息的操作和访问。

继承(Inheritance)

继承就是指现有一个类，在无需重新编写原来的类下扩展新的功能。通过继承创建的类称为“子类”或“派生类”，被继承的类称为“父类”或“基类”。继承的方式有：实现继承和接口继承。

实现继承关键字extends public class A extents B{} 意为类A继承了类B，A就是B的子类。
接口实现关键字implements public class A implements B{} 意为类A实现了接口B
实现继承可以直接使用父类的属性与方法而无需而外编码。
接口继承是指仅使用接口属性和方法的名称，但子类必须提供方法的实现。

多态(Polymorphism)

多态就是指同一个行为具有多个不同表现形式或形态的能力，简单的说就是不同的对象执行相同的操作会有不同的结果。

最常见的多态就是将子类传入父类参数中，运行时调用父类方法时通过传入的子类决定具体的内部结构或行为。

面向对象的五大基本原则

单一职责原则(Single-Responsibility Principle)

指一个类最好只做一件事，类的功能要单一，职责过多，可能引起它变化的原因就越多，这将导致职责依赖，相互之间就产生影响，从而大大损伤其内聚性和耦合度。

依赖倒置原则(Dependecy-Inversion Principle)

高层模块不依赖于底层模块，二者都同依赖于抽象。因为抽象是不变的，抽象的稳定性决定了系统的稳定性，依赖于抽象，就是对接口编程，不要对实现编程。

开放封闭原则(Open-Closed principle)

指一个模块在扩展性方面应该是开放的，而在更改性方面应该是封闭的。
对扩展开放，意味着有新的需求或变化时，可以对现有代码进行扩展，以适应新的情况

对修改封闭，意味着类一旦设计完成，就可以独立完成其工作，而不要对其进行任何尝试的修改。

里氏替换原则(Liskov-Substitution Principle)

指只要父类能出现的地方子类就可以出现，而且替换为子类也不会产生任何错误或异常，使用者可能根本就不需要知道是父类还是子类。但是，反过来就不行了，有子类出现的地方，父类未必就能适应。

接口隔离原则(Interface-Segregation Principle)

指使用多个小的接口，而不要使用大的接口。不要强迫类去依赖它用不到的接口或方法，一个类对另外一个类的依赖应该建立在最小的接口上。使用多个专门的接口比使用单个接口要好的多。
